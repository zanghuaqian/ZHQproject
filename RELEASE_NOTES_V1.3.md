# Excel Transformer Skill v1.3.0 发布说明

发布日期：2026-01-30

## 🎯 本次更新重点

本次更新主要修复了跨月退订单的显示和计算问题，优化了退款金额的智能判断逻辑。

## ✨ 新增功能

### 智能跨月退款计算
- 从交易对账单的"订单金额(元)"字段自动获取退款金额
- 智能判断退款类型：
  - **仅退快递费**：当订单金额 = 快递费时，退款-快递费填充快递费，退款-退货为0
  - **退快递费+退货**：当订单金额 = 快递费 + 实付款时，分别填充快递费和实付款

### 退款字段优化
- 退款-快递费和退款-退货始终显示为正数（不带负号）
- 钱包款自动计算为负数，显示为红色括号

## 🔧 Bug修复

### 跨月退订单收款金额问题
- **问题**：跨月退订单的收款金额被公式覆盖，未正确显示为0.00
- **修复**：跨月退订单的收款金额列不设置公式，保持原值0.00

### 跨月退订单钱包款格式问题
- **问题**：跨月退订单的钱包款负数格式被覆盖，未显示为红色括号
- **修复**：在设置数字格式时，跳过跨月退款行的V列，保持红色括号格式

### 空行格式问题
- **问题**：跨月退订单和普通订单之间的空行被填充了公式和格式
- **修复**：空行不再填充任何数值、公式和格式

## 🚀 使用方法

### 安装
将 `excel-transformer.skill` 文件复制到你的技能目录即可使用。

### 依赖安装
```bash
pip install pandas openpyxl
```

### 基本用法
```bash
python scripts/generate_statement.py <原数据文件> <交易对账单文件> <月份> [输出文件]
```

### 跨月退款支持
```bash
python scripts/generate_statement.py <原数据文件> <交易对账单文件> <月份> [输出文件] --cross-month 202512=<12月原数据文件>
```

## 📝 示例

```bash
# 生成1月对账单
python scripts/generate_statement.py \
    "E:\Downloads\20260128_100920.xlsx" \
    "E:\Downloads\自定义交易对账单_44128319_20260101000000_20260128235959.xlsx" \
    1

# 生成1月对账单（包含12月跨月退款）
python scripts/generate_statement.py \
    "E:\Downloads\20260128_100920.xlsx" \
    "E:\Downloads\自定义交易对账单_44128319_20260101000000_20260128235959.xlsx" \
    1 \
    --cross-month 202512=E:\Downloads\20260104_094310.xlsx
```

## 💡 技术细节

### 跨月退款计算逻辑
1. 从交易对账单获取退款金额（订单金额(元)字段）
2. 从原数据获取快递费和实付款
3. 比较退款金额与快递费、实付款的关系：
   - `abs(退款金额 - 快递费) < 0.01` → 仅退快递费
   - `abs(退款金额 - (快递费 + 实付款)) < 0.01` → 退快递费和退货
4. 根据判断结果填充对应字段

### 格式设置优化
- 跨月退款行的收款金额（R列）不设置公式
- 跨月退款行的钱包款（V列）不覆盖红色括号格式
- 空行不设置任何公式和格式
